# 🗑️ Sistema de Trazabilidad de Residuos Patológicos

> **Full-stack enterprise application** for pathological waste traceability management, developed for AESA Misiones S.A. (VEOLIA concession). Covers the complete waste lifecycle: from generator registration to treatment certification.

---

## 🧩 Overview

This system manages the traceability of pathological waste generated by hospitals, clinics, veterinary offices, laboratories, and other healthcare facilities in Misiones, Argentina. It replaces manual paper-based processes with a digital platform that ensures regulatory compliance (Ley 24.051).

The platform connects four types of users — administrators, clients/generators, collection employees, and treatment employees — each with a dedicated dashboard and role-based access control via JWT.

## ✨ Key Features

### 👤 Generator / Client Portal
- Self-registration with contract auto-generation (PDF via iText)
- Bag purchasing flow integrated with **MercadoPago** payment gateway
- Withdrawal scheduling with working-day calendar picker
- Purchase history and bag inventory tracking

### 🛡️ Admin Panel
- Generator approval workflow with automated email sending
- Route sheet creation assigning orders → employees → vehicles per zone
- Real-time inventory (stock) management with low-stock alerts and auto-email restock requests
- Three analytical report types (by zone, by sales, by treatment) with Excel/PDF export

### 🚛 Collection Employee App
- Mobile-friendly dashboard showing assigned daily routes
- **QR code scanner** for individual seal validation and collection tracking
- Map integration with real route calculation via OSRM (OpenStreetMap Routing Machine)
- Route optimization using Nearest Neighbor algorithm (geocoded via LocationIQ)
- Route closure form with km and return time registration
- Offline-tolerant GPS with automatic fallback to default location

### 🧪 Treatment Employee App
- Container management with weight-based capacity control
- Batch or individual seal weighing before container assignment
- Container close → treatment → reuse cycle
- Automatic **daily certificate generation** (PDF) sent by email to each generator

### 📊 Reporting Engine
| Report | Filters | Export |
|--------|---------|--------|
| Zones (orders by area) | Date range, status, zone | Excel + PDF |
| Sales (bags by generator) | Date range, type, zone | Excel + PDF |
| Treatment (treated seals) | Date range, type, zone | Excel + PDF |

## 🔐 Security & Authentication

- JWT-based stateless authentication
- Role-based route guards: `ADMIN`, `CLIENTE`, `EMPLEADO`, `EMPLEADO_TRATADOR`
- BCrypt password encoding
- CORS configured for local development

---

## 🗺️ Geolocation & Route Optimization

- Address geocoding via **LocationIQ** with intelligent Córdoba-bounded validation
- Fallback coordinates generated deterministically from address hash
- Route optimization using Nearest Neighbor algorithm
- Real driving routes via **OSRM** (free, no API key required)
- Live GPS tracking with permission-aware error handling

---

## 💳 Payments

- **MercadoPago** Checkout integration (sandbox-ready)
- Preference creation with item, price, and quantity
- Automatic sale confirmation via webhook or polling on window close
- Seal state changes triggered on payment approval: `DISPONIBLE → OCUPADO`

---

## 📧 Email Automation

- New generator registration → contract PDF attached (iText-generated)
- Generator approval → temporary password delivered
- Low stock detection → procurement alert email
- Daily batch job (23:00) → treatment certificates sent to all generators treated that day

## 🚀 Getting Started

### Prerequisites
- Java 21+
- Node 18+ / npm
- MySQL 8+

### Backend

```bash
# Clone and navigate
cd Back

# Configure application.properties
# - spring.datasource.url
# - jwt.secret
# - mercadopago.access.token
# - spring.mail.*

# Run
./mvnw spring-boot:run
```

### Frontend

```bash
cd Front
npm install
ng serve
# → http://localhost:4200
```

## 🎯 Domain Workflow

```
Generator registers → Admin approves → Generator buys bags
       ↓
Seals assigned (DISPONIBLE → OCUPADO)
       ↓
Admin creates route sheet → Employee collects (QR scan)
       ↓
Seals: OCUPADO → RECOLECTADO → EN_PLANTA
       ↓
Treatment employee weighs + adds to container → closes container
       ↓
Seals: EN_TRATAMIENTO → TRATADO
       ↓
Certificate PDF auto-generated and emailed to generator
```
